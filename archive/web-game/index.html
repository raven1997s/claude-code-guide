CLI 参数完整参考

Claude Code 支持丰富的命令行参数，用于控制会话行为

基础用法

claude [options] [command] [prompt]

启动模式

参数 简写 说明
--print -p 打印响应后退出（非交互模式）
--continue -c 继续最近的对话
--resume [id] -r 恢复会话，可选指定会话 ID
--fork-session 恢复时创建新会话而非复用原 ID

示例

# 非交互模式
claude -p "列出当前目录文件"

# 继续上次对话
claude -c

# 恢复特定会话
claude -r abc123-def4-5678

模型与 Agent

参数 说明
--model <model> 指定模型（sonnet/opus/haiku 或完整名称）
--agent <agent> 使用特定 Agent
--fallback-model <model> 当主模型过载时的备用模型

示例

# 使用模型别名
claude --model sonnet
claude --model haiku

# 使用自定义 Agent
claude --agent reviewer "审查这段代码"

# 设置备用模型
claude --model sonnet --fallback-model haiku

权限与安全

参数 说明
--permission-mode <mode> 权限模式（default/acceptEdits/bypassPermissions）
--allowed-tools <tools...> 允许的工具列表
--disallowed-tools <tools...> 禁止的工具列表

示例

# 自动接受编辑（适合开发环境）
claude --permission-mode acceptEdits

# 只允许读取操作
claude --allowed-tools Read,Glob,Grep

# 禁止 Bash 工具
claude --disallowed-tools Bash

更多参数

  • --version / -v - 输出版本号
  • --help / -h - 显示帮助
  • --debug [filter] - 启用调试模式
  • --max-budget-usd <amount> - 最大 API 调用花费
  • --output-format json - JSON 输出格式

内置斜杠命令

在 Claude Code 交互式会话中使用的命令

/help

基础

显示帮助信息和可用命令

> /help

# Claude Code 命令参考
#
# /help      - 显示此帮助
# /clear     - 清除对话历史
# /commit    - 创建 Git 提交
# /tasks     - 查看任务列表
# /exit      - 退出

/clear

基础

清除当前对话的上下文和记忆

> /clear

# 对话上下文已清除

使用场景：

  • 开始新话题
  • 重置对话状态
  • 清除错误上下文

/commit

Git

创建 Git 提交，Claude 会自动分析改动并生成提交信息

> /commit

# Claude 会：
# 1. 运行 git status 查看改动
# 2. 运行 git diff 查看具体变更
# 3. 分析改动内容
# 4. 生成符合规范的提交信息
# 5. 执行 git add 和 git commit

/tasks

管理

查看和管理当前会话的任务列表

> /tasks

# 当前任务：
#
# [ ] 1. 分析现有代码结构
# [ ] 2. 设计新的 API 接口
# [ ] 3. 实现用户认证
# [ ] 4. 编写单元测试

/exit

基础

退出 Claude Code 会话

> /exit

# 或者使用快捷键：
# Ctrl+D (Unix/Linux/macOS)
# Ctrl+Z 然后 Enter (Windows)

会话保存：

  • 默认自动保存会话
  • 可通过 --no-session-persistence 禁用
  • 使用 claude -c 继续上次会话

/editor

高级

打开默认编辑器编辑长消息

> /editor

# 打开 $EDITOR 或 vim 编辑器
# 编辑完成后保存退出

使用场景：

  • 编写长提示词
  • 多行代码输入
  • 复杂问题描述

/continue

高级

继续生成上次被截断的响应

> /continue

# 继续上次的输出...

/skip

控制

跳过当前正在执行的工具调用

使用场景：

  • 工具执行时间过长
  • 不需要当前操作结果

/accept

权限

接受所有待处理的权限请求

> /accept

# 批准所有挂起的操作

/reject

权限

拒绝所有待处理的权限请求

> /reject

# 拒绝所有挂起的操作
